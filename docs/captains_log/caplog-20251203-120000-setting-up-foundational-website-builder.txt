## Decisions Made
--------------------------------------------------------------------------------

1. **Astro 5 + Tailwind CSS 4** - Modern stack matching ex10x architecture,
   excellent DX and performance

2. **Centralized site config** - All client-specific info in one file
   (src/config/site.ts) for easy customization

3. **CSS custom properties for theming** - Primary colors defined in global.css
   @theme block for easy brand customization

4. **Content collections for blog** - Type-safe, validated blog posts using
   Astro's built-in content collections

5. **Component-based architecture** - Reusable components (Hero, Section, CTA,
   Card) that can be composed for any page layout

6. **Vertical taxonomy structure** - Organized industries by profitability and
   demand (Tier 1: Home Services, Construction, Legal, Medical)

## Mistakes & Lessons
--------------------------------------------------------------------------------

1. **@apply in Astro scoped styles** - Tailwind CSS 4 with Vite plugin doesn't
   support @apply in component-scoped <style> blocks. Use utility classes
   directly or put @apply rules in global.css.

2. **Import validation** - Always check that variables used in templates are
   properly imported in the frontmatter, especially when copying patterns
   between files.
