================================================================================
DECISIONS MADE
================================================================================

1. **Obscure path over authentication** - Used leetspeak path (`d4t4/r3p0rts`)
   rather than requiring admin key authentication. Simpler, and reports aren't
   sensitive data.

2. **Dual storage** - Keep archive in `docs/reports/` and served copy in
   `public/d4t4/r3p0rts/`. Archive stays in git, public copy gets deployed.

3. **Auto-detect report type** - Rather than manual metadata, detect type from
   filename keywords (seo, traffic, performance, audit).

4. **Both noindex methods** - Used both `<meta robots>` tag AND robots.txt
   disallow for defense in depth.

================================================================================
LESSONS LEARNED
================================================================================

1. **Always verify git status before "deploying"** - The deploy commands pull
   from origin, so uncommitted local changes won't appear on servers.
