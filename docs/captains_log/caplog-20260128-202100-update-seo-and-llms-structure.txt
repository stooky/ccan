## 1. robots.txt - AI Crawler Permissions

**File:** `public/robots.txt`

**Changes:**
- Explicitly ALLOW all major AI crawlers
- Added 10+ AI bot user-agents with Allow: /
- Documented why (increases citation probability)
- Blocked /api/ and /data/ directories

```
User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /
...
```

**Why:** Many sites block AI crawlers by default. Explicitly allowing them signals
the content is available for training/citation.

--------------------------------------------------------------------------------

## 2. Enhanced Meta Robots Tag

**File:** `src/layouts/Layout.astro`

**Before:**
```html
<meta name="robots" content="index, follow" />
```

**After:**
```html
<meta name="robots" content="index, follow, max-snippet:-1, max-image-preview:large, max-video-preview:-1" />
```

**Why:** Tells search engines to use unlimited snippet length and large images in
results, improving rich result appearance.

## 3. Enhanced LocalBusiness Schema

**File:** `src/layouts/Layout.astro`

**Added Properties:**
- `foundingDate`: "2022"
- `founder`: { "@type": "Person", "name": "Sam" }
- `areaServed`: Array of cities (Saskatoon, Martensville, Warman, Regina, etc.)
- `knowsAbout`: ["Shipping Containers", "Storage Containers", "Sea Cans", etc.]
- `slogan`: "Saskatchewan's Premier Seacan Solutions Provider"

**Why:** E-E-A-T signals (Experience, Expertise, Authority, Trust). More properties
= better entity understanding by LLMs.

## 7. Sitemap Priority Configuration

**File:** `astro.config.mjs`

**Added serialize function with priority weighting:**
```javascript
sitemap({
  serialize(item) {
    if (item.url === 'https://ccansam.com/') {
      item.priority = 1.0;
      item.changefreq = 'weekly';
    }
    else if (item.url.includes('/seacan-storage-containers-for-sale')...) {
      item.priority = 0.9;
    }
    // ... more rules
  }
})
```

**Priority Levels:**
- 1.0: Homepage
- 0.9: Main service pages (sales, rentals, on-site)
- 0.85: City landing pages
- 0.8: Individual container pages
- 0.7: About, Contact
- 0.6: Blog posts
- 0.3: Terms, Privacy
- 0.5: Default

**Why:** Signals relative importance of pages to search engines.

================================================================================
DECISIONS MADE
================================================================================

1. **Explicit AI Allow vs implicit** - Most bots crawl regardless, but explicit
   permission signals consent and may increase citation probability

2. **Speakable with CSS selectors** - More maintainable than XPath

3. **City pages before content** - Local SEO ROI higher than generic blog content

4. **Regina before Moose Jaw** - Larger population = more search volume

5. **Table over grid for comparisons** - Semantic HTML more extractable by LLMs

6. **Priority 0.85 for city pages** - Between services (0.9) and products (0.8)

================================================================================
LESSONS LEARNED
================================================================================

1. **AI search is different** - Traditional SEO focuses on clicks; AI SEO focuses
   on being the source that gets cited (often no click)

2. **Schema matters more for LLMs** - Structured data is easier to parse than
   natural language for AI models

3. **Definitive statements get quoted** - Vague language doesn't get cited

4. **robots.txt AI directives are new** - Many sites haven't updated for AI bots

5. **City pages are quick wins** - Template-based, high local impact

================================================================================
MISTAKES TO AVOID
================================================================================

1. Don't block AI crawlers unless you have specific legal/privacy reasons

2. Don't use generic meta descriptions - each page needs unique, specific text

3. Don't forget to update city names in PlaceSchema when adding new pages

4. Don't hardcode coordinates - use the cityCoordinates lookup in PlaceSchema
