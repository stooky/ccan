## Summary

Parameterized the entire build/deploy pipeline for N-site deployment. The
codebase can now build and deploy different sites from the same repo using
SITE_DIR, SITE_URL, and SITE_ROOT environment variables.

## Current State

The site now supports multi-site via a `sites/<domain>/` directory pattern.
config.yaml at the root is a symlink to `sites/ccansam.com/config.yaml`.
All PHP files use a shared `_config.php` that reads SITE_ROOT from nginx.
deploy.js reads from a `sites.yaml` manifest and includes a post-deploy
health check.

## DECISIONS MADE

1. **Symlink over copy** — Root config.yaml is a symlink to sites/ccansam.com/
   config.yaml, not a copy. This means one source of truth, no sync needed.

2. **Shared PHP with env var** — PHP files use SITE_ROOT env var (set by nginx)
   rather than per-site PHP-FPM pools. Simpler for solo dev, per-site pools
   deferred to when >5 sites on one server.

3. **Build on server (for now)** — Still runs npm run build on the VPS. Rsync
   of dist/ deferred to Phase 3 when server count increases.

4. **Backward compatible** — All changes fall back to current behavior when no
   env vars are set. Zero breaking changes for existing deployment.

ISSUES ENCOUNTERED
================================================================================

## 1. SSH Key with Space in Filename

**Symptom:** All SSH connections failed — "No such file or directory"
**Cause:** Key was named "id_rsa 1" (with space), permissions were 644
**Resolution:** chmod 600, quoted path in scripts

## 2. CSP Blocked Chart.js CDN

**Symptom:** SEO report page rendered blank below the stat cards
**Cause:** New CSP headers blocked cdnjs.cloudflare.com
**Resolution:** Added cdnjs.cloudflare.com to script-src allowlist

## 3. GitHub Push with Wrong Identity

**Symptom:** Permission denied pushing to stooky/ccan
**Cause:** Remote was HTTPS, authenticating as msifoss
**Resolution:** Switched remote to SSH with github-stooky host alias

================================================================================
LESSONS LEARNED
================================================================================

1. **Always test CSP with real page content** — curl -I only checks headers,
   doesn't reveal blocked resources. Load the actual page.

2. **Check SSH key names carefully** — spaces in filenames are legal but
   break unquoted paths in scripts.

3. **Git remote identity matters** — HTTPS remotes use default credential,
   SSH remotes use host-specific keys from ~/.ssh/config.
